# UI UX Pro Max

A searchable database of UI styles, color palettes, font pairings, chart types, and UX guidelines to help you write better code.

## Instructions

Whenever you are asked to design a UI, build a frontend, or make something "look good", you MUST follow this process:

1.  **Analyze the Vibe**: Determine if the user wants "Corporate", "Creative", "Cyberpunk", "Minimalist", etc.
2.  **Select a Palette**: Choose 3-4 colors. 1 Primary, 1 Secondary, 1 Background, 1 Text.
3.  **Typography**: Choose a pairing. (e.g., Inter + Roboto Mono).
4.  **Components**:
    - **Cards**: Use shadows or borders appropriately.
    - **Buttons**: distinct primary/secondary styles.
    - **Spacing**: Use a 4px or 8px grid system.
5.  **Output**: Write the CSS/Tailwind code directly.

## Rules

- **No Placeholders**: Never use `lorem ipsum` or grey boxes. Generate real content.
- **Accessibility**: Ensure high contrast ratios.
- **Mobile First**: Always design for narrow screens first.
