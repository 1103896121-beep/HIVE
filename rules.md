# Antigravity 通用工程规则 (Rules)

每次交互或代码生成必须严格遵守以下约定。

## 一、语言与输出约定
- 所有回复、思考过程、计划、方案、说明、注释、文档 **必须使用简体中文**
- 代码中的标识符保持英文，不使用拼音
- 错误信息、日志内容允许为英文

## 二、技术默认约定
- **前端**：默认使用 React + TypeScript
- **后端**：默认使用 Python (优先 FastAPI)
- 若无特殊说明，均遵循以上技术选型。

## 三、通用代码规范

### 命名规范
- **变量/函数**：`camelCase`
- **类/组件**：`PascalCase`
- **常量**：`UPPER_SNAKE_CASE`
- **文件/文件夹**：`kebab-case`
命名应语义清晰，禁止随意缩写。

### 注释规范（强制）
- 注释用于解释【为什么这样设计】，而不是代码字面含义
- 复杂逻辑、业务判断、边界条件必须写注释
- **禁止**无意义注释
- **禁止**硬编码
- 如果项目涉及到国际化，**必须**在写页面信息前先进行国际化(i18n)处理

统一注释标记：
- `TODO`: 待实现功能
- `FIXME`：已知问题或潜在缺陷
- `NOTE`：重要设计说明
- `HACK`：临时方案，后续必须重构

#### 函数注释规范
- 前端 (JSDoc): 用 `/** */` 包裹，说明 `@param` 和 `@returns`
- 后端 (Python Docstring): 必须写在函数定义下方第一行，使用 `""" """`

## 四、前端规范 (React)
- 使用**函数组件**，不使用类组件
- 单个组件只承担单一职责，展示逻辑与业务逻辑分离
- 复用逻辑必须抽离为自定义 Hook (必须以 `use` 开头)
- **Hooks 使用规范**: 只能在函数组件或自定义 Hook 顶层调用，不允许在条件、循环中调用。一个 Hook 只处理一种职责。
- **Props 规范**: 必须使用 TypeScript 类型定义，使用解构方式接收 props，非必传参数使用 `?`。
- **性能**: 避免不必要重复渲染，合理使用 `useMemo`/`useCallback`，列表渲染必须提供稳定的 key，长列表使用虚拟滚动，路由/组件支持懒加载。

## 五、后端规范 (Python)
- 必须使用 Python >= 3.10
- 优先使用 FastAPI
- 所有函数与方法必须标注类型指引 (Type Hints)
- **禁止使用裸 `except:`**
- **禁止使用 `print`** 作为业务日志方式，必须使用标准 `logging` 模块
- 编写新函数/方法前，必须**先检查**是否有现成的重复方法，防止代码冗余

### 分层结构（严格遵守）
- `api`: 请求解析与响应封装。禁止在 api 层直接操作数据库。
- `service`: 业务逻辑处理，权限校验。
- `repository`: 数据库访问层。
- `schema`: 请求/响应数据 Pydantic 校验。
- `model`: ORM 模型定义。

## 六、安全规范（重点）
- 永远不信任客户端输入，所有输入必须进行校验
- 敏感操作必须经过身份与权限校验 (Service 层)
- 前端防 XSS / CSRF 攻击，前端禁止使用 `dangerouslySetInnerHTML`
- 不在前端明文存储敏感信息，Token 推荐使用 HttpOnly Cookie (或按项目架构规范)
- 后端使用 Pydantic 进行参数校验
- 所有密钥配置必须从环境变量(或 `.env`)中读取 (如 `os.getenv("SECRET_KEY")`)
- 敏感字段返回前端前须脱敏
- 密码等敏感数据必须加密哈希存储

## 七、部署规则 (AI 专用・本地 Git 版)
核心前提：
- 服务器已预装 Docker、Git。
- 源码必须由 Git 管理，无遗漏。
- 用 Dockerfile + .dockerignore 精准控制镜像。
- 一键部署脚本：封装「拉取代码 → 构建镜像 → 启动容器」全流程。
- 使用 `.env` 隔离环境。

部署流程（按顺序执行）：
1. 每次代码变更后：本地打包 (基于 `git archive`)
2. 将环境和代码推送到部署服务器。
3. 若首次/需更新配置：生成相应的 `Dockerfile`, `.dockerignore`, `deploy.sh`。
4. 部署服务器解压代码并执行 `deploy.sh`。
5. 若部署失败则通过历史镜像快速回滚。
6. 最后使用 Docker 命令校验容器运行状态。
