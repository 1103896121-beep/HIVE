# MerchLens AI 生产环境配置指南

本文档全面梳理了 MerchLens AI 项目在生产环境（Production）下的部署细节、架构分布以及关键配置项。

## 1. 基础服务器信息 (Host Info)
*   **服务器 IP**: `103.91.219.230`
*   **操作系统**: Linux (Ubuntu 体系)
*   **项目根目录**: `/opt/merchlens_ai_work`
*   **网关代理**: 外部暴露采用 Nginx Proxy Manager (`npm-app-1` 容器) 进行 SSL 证书挂载及域名转发。
*   **自动化部署**: 采用独立脚本 `master_deploy_legal_v5.py` 进行一键代码推包、解压及热更。

## 2. Docker 容器架构 (Containers)
生产环境由 `/opt/merchlens_ai_work/docker-compose.prod.yml` 编排管理，共包含 5 个核心微服务容器：

*   **`merchlens_db`**
    *   **镜像**: `postgres:15-alpine`
    *   **职责**: 提供结构化业务数据持久层（任务、用户、账单等历史）。
    *   **宿主机端口**: 未向外暴露，仅限内网通讯（5432）。
*   **`redis`**
    *   **镜像**: `redis:7-alpine`
    *   **职责**: 为 Celery 任务队列、高频缓存、SSE 状态分发提供内存型缓存。
    *   **宿主机端口**: 未向外暴露。
*   **`backend`**
    *   **引擎**: Python FastAPI / Uvicorn (Port 8000)
    *   **职责**: 主逻辑 API 服务器，处理图像合成、VTON 排片逻辑等核心计算请求。
    *   **高可用机制**: 配置了严格的 Healthcheck (`curl -f http://localhost:8000/health`)，确保 AI 模型初始化完成后再接入网关流量。
*   **`worker`**
    *   **引擎**: Python Celery (`app.core.celery_app worker`)
    *   **职责**: 利用相同的 backend 镜像运行，专门处理重度耗时的异步后台任务（如下载压缩包生成）。
*   **`frontend`**
    *   **引擎**: Nginx 纯静态服务器
    *   **职责**: 解析映射并托管 React 单页应用 (SPA)。
    *   **宿主机端口映射**: 宿主机的 `5173` 映射至容器内 Nginx 的 `80`。

## 3. 存储卷分配 (Volumes)
所有核心用户数据及素材被安全挂载为 Docker Volumes，确保服务器重启/更新时不丢失：

*   **数据库存储**:
    *   **`postgres_data`**: 映射到 DB 容器内的 `/var/lib/postgresql/data`。
*   **后端文件及生成记录**:
    所有后端静态卷被同时挂载给 `backend` 和 `worker` 两个容器以实现磁盘互通处理：
    *   **`static_images`**: `/app/static/images` 原素材存储区。
    *   **`static_generations`**: `/app/static/generations` AI 渲染成片存储区。
    *   **`static_vton_assets`**: `/app/static/vton-assets` 虚拟试衣相关的底层素材。
    *   **`storage_data`**: `/app/storage` 合成营销图的落地区。

## 4. 路由配置 (Nginx & Gateway)
前端 Nginx 配置文件位于 `frontend/nginx.conf`，涵盖了关键的路由转发逻辑：

*   **HTML 强刷机制**: React 单页应用入口 (`/index.html`) 被严格限制了 `Cache-Control "no-store, max-age=0"`，彻底阻断了 CDN/浏览器对最新前端代码包的缓存。
*   **API 反向代理**:
    所有以 `/api/v1` 和 `/api` 开头的请求均被 proxy_pass 直接打到局域网的后端的 `http://backend:8000` (上游 `backend_api`)。
*   **连接保护套件**:
    *   配置了 **120s 的超长超时** (`proxy_read_timeout 120s`) ，以保障 AI 生图和图片下载长轮询时不被代理杀断。
    *   关闭了请求和响应缓冲 (`proxy_request_buffering off`)，针对流式响应 (SSE event-stream) 做了最佳优化。
*   **静态服务器回源**:
    类似 `/static/` 的图片路径，由 Nginx 直接回源到 `backend:8000` 将图片透写回前端。

## 5. 编译环境与 NPM 配置 (Frontend Build)
*   **构建工具**: `Vite` + `React` + `TypeScript`
*   **生产构建指令**: `npm run build`。该指令在本地机器执行，成功后将生成的 `dist` (或静态资源) 以 `.tar.gz` 打包通过 `master_deploy_legal_v5.py` SFTP 推送至远端并覆写至 `/opt/merchlens_ai_work/frontend/`。
*   **环境变量加载**: 本地需确保包含必要的 `.env` 用于在打包时向硬编码 API 注入对应路径。生产服务器上，后端的环境变量则统一受控于 `/opt/merchlens_ai_work/backend/.env.prod`。
