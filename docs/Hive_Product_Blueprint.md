# Hive - 沉浸式在线自律陪伴 App 🐝
**产品定位**: 面向苹果(iOS)生态的高级感、克制、沉浸的“线上共同自律室”。
**核心slogan**: 同处蜂巢，各自闪能。 (Together in the Hive, Shining Alone.)

---

## ▎一、 视觉与交互风格基调 (iOS 专属高级感)
苹果用户极其看重 **“少即是多 (Less is More)”** 与 **“物理直觉学交互”**。Hive 的设计必须彻底抛弃市面上常见的幼稚卡通风或老旧的打卡UI。

1. **色彩哲学 (Color Palette)**
   - 全局背景使用极度克制的 **纯黑 (Deep AMOLED Black)** 与 **钛灰 (Titanium Gray)**，不仅省电且在夜间自习时护眼。
   - 核心激励色（如蜂格点亮、达成目标）使用 **琥珀金 (Amber Gold)** 与 **荧光薄荷 (Neon Mint)**，形成强对比的高级科技感。
2. **材质与动效 (Material & Motion)**
   - 大量运用 **Background Blur (高斯模糊毛玻璃)**，呼应 iOS 原生质感。
   - 抛弃生硬的界面跳转。专注倒计时圈采用类似于 Apple Watch "呼吸环" 的丝滑 60fps 动效。
3. **触觉反馈 (Haptics Haptic Touch)**
   - 在用户点击“开始专注”、拖拽调解时间、或同伴给你发送鼓励时，调用手机马达提供清脆的 `Taptic Engine` 物理震动反馈，增强产品的高级触感。

## ▎二、 核心 MVP 功能蓝图 (The Core Features)

### 1. The Hive (蜂格自习室)
*不再是干瘪的头像列表，而是一个具备空间感的数字蜂巢。*
*   **六边形矩阵阵列**：进入大厅，用户看到的是由无数半透明六边形网格组成的“蜂巢界”。每个六边形代表一个正在自律的用户。
*   **非侵入式社交陪伴**：
    *   **呼吸灯状态**：网格在微微呼吸频闪，代表该用户正在专注中。
    *   **科目共鸣**：例如学习“考研英语”的人，格子边缘泛起极其微弱的特定颜色光晕。
*   **小队包间 (Squad Room)**：支持 4-6 个熟人组队私密包间。只要大家一起努力，整个小队的蜂巢就能“采蜜”升级。

#### 1.1 首页匹配逻辑 (Smart Matching)
首页蜂窝中显示的人遵循 **「自动匹配 + 锚定好友」** 混合策略：

**匹配公式：** 首页蜂窝 = Bond 搭子（固定席位）+ 系统匹配的同路人（流动席位）

1. **Bond 搭子优先占位**：已绑定的 Bond 好友如果在线，永远固定出现在蜂窝核心位置。
2. **流动席位自动补齐**：剩余位置由系统根据优先级匹配：**同学科 > 同地区 > 同时段 > 随机**。
3. **社交关系自然生长**：与同一陌生人连续 3 天同房间 → 系统提示“是否绑定为 Bond？”

#### 1.2 分层展示架构 (Scale Display)

| 层级 | 名称 | 人数 | 展示方式 |
|------|------|------|----------|
| L1 | Squad（小组） | 4~12 人 | 首页蜂窝网格，完整头像+状态 |
| L2 | Hive（蜂巢/房间） | 50~200 人 | 蜂窝外围微缩光点环 |
| L3 | Region（地区） | 1000+ 人 | 数字统计 + 活跃度指示条 |
| L4 | Planet（全球） | 10000+ 人 | 3D 真实地球模式 |

#### 1.3 羁绊系统 (Digital Bonds)
好友系统定义为 **“数字羁绊 (Bond)”**，原则：“看得见彼此奋斗，却说不了废话”。

- **禁止私聊**：专注期间好友之间严禁发送文字。
- **仅状态可见**：只能看到好友蜂格是否点亮和正在努力的科目。
- **轻推 (Nudge)**：给正在专注的好友发一个模拟 Taptic Engine 的微震动。
- **共同契约 (Joint Pact)**：与好友发起“连带责任”任务，一人分心则双方受罚。
- **24h 对比**：仅展示过去 24 小时内彼此的专注总时长，减少长期攀比。

### 2. Focus Lock (深度强制专注引擎)
*   **白噪音沉浸隧道**：从潮汐海浪、图书馆翻书到火星风暴，专注开始时，UI界面渐暗，声音渐亮。
*   **严苛模式 (Strict Mode)**：利用 iOS 原生的权限管理，一旦设置了锁机，系统将彻底拦截其他应用通知（如进入“勿扰模式”）。中途切出 App （比如回微信）将直接导致本次记录失败，并且你的"蜂格"会黯淡，全组人都能看见你的“分心”。

### 3. Hex-Stats (全息成就统计)
*   **六边形拼接的数据可视化**：抛弃传统的柱状图。你每天完成的专注，将会生成真实的“金色蜂巢格子”。坚持一年，你的个人主页将是一面极其壮观的、由几百个发光的黄金六边形组成的立体画卷，极具成就分享欲（完美适配朋友圈分享）。

### 4. Identity System (全息身份系统)
*不仅是一个账户，而是你在虚拟蜂巢中的真实自律化身。*
- **呼吸感头像**：头像外圈整合了实时的专注状态灯（Live Status），与蜂巢物理状态同步。
- **Sparks 能量值**：通过深度专注换取的数字货币，用于解锁高级主题或特殊蜂格光效。
- **自适应主页**：支持极简化的资料编辑与头像上传，设计调性保持 iOS 级别的克制与高级感。

## ▎三、 技术架构蓝图 (Technical Architecture)

为了保证最极致的 iOS 体验，并以极高的开发效率完成，推荐以下技术栈组合：

### 1. 前端/客户端 (Client-Side)
- **框架推荐**：**React Native + Expo** (当前市面上兼具效率和高性能原生的跨平台最强方案之一)。如果不考虑安卓拓展，直指性能巅峰，也可在后期转向纯 **SwiftUI**。
- **UI 组件化**：完全基于 `Tailwind CSS`(通过 `NativeWind` 转译 Native) 进行像素级还原，配合 `Reanimated` 构建最高级别的丝滑手势与 60帧动画。
- **音视频流**：集成轻量级跨平台音频引擎播放环境白噪音，可根据网络状态预加载资源。

### 2. 后端服务端 (Server-Side)
- **核心框架**：**Python - FastAPI**。
  - 原因：FastAPI 自带对 WebSocket 的极好支持。因为我们需要在用户中途切出 App、或某一个人的番茄钟结束时，将状态变化“实时”推送给同个蜂巢包间里的其他人，WebSocket 长连接是刚需。
- **持久化层**：**PostgreSQL** + `SQLAlchemy`。储存海量的时间日志、用户关系和分析报表。
- **缓存层**：**Redis**。因为用户的在线状态（灯亮、灯灭）和组团信息处于高并发变化中，必须通过 Redis 在内存中进行高速读写和发布订阅 (Pub/Sub)。

### 3. 设计规范工作流 (Design Workflow)
*在真正写第一行代码前，我们要明确的设计流程：*
1.  确立 Hive 的 **Design Tokens**（包含具体颜色的 HEX 值、圆角幅度、字体等）。
2.  输出纯黑色调的 **高保真交互视觉稿图** (Figma 规范)。
3.  通过 React Component 将这些视觉转化为跨平台的重构组件库。
